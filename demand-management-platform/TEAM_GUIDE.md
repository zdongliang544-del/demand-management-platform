## 四、功能说明

### 4.1 需求列表

**入口**：左侧菜单 → 📋 需求列表

**功能**：
- 查看所有需求
- 按条件筛选需求
- 搜索需求标题
- 导出 Excel

**字段说明**：

| 字段 | 说明 |
|------|------|
| 编号 | 系统自动生成，格式 DMxxxx |
| 标题 | 需求名称，限 30 字 |
| 需求内容 | 详细描述，列表中显示前 20 字 |
| 类型 | 平台 / 业务 |
| 端 | APP / KILI / Web（多选） |
| 关联模块 | 14 个模块选项（多选） |
| 优先级 | P0（紧急）/ P1（重要）/ P2（一般） |
| 状态 | 待确认 / 待评审 / 开发中 / 测试中 / 已发布 / 已关闭 / 挂起 |
| 产品PM | 负责人（多选） |
| 已应用国家 | 需求已上线的国家（多选） |
| 登记信息 | 登记人和时间 |
| 能力形式 | 标准能力新增/迭代、差异能力新增/迭代等 |
| 审查人 | 最后审查的用户和时间 |

### 4.2 新建需求

**入口**：需求列表页 → ➕ 新建需求

**必填字段**：
- 类型、端、角色、关联模块
- 已应用国家、需要应用国家
- 需求标题、需求内容
- 当前状态、优先级、产品PM

**选填字段**：
- 需求价值（限 100 字）
- 需求提出国家、提出人
- 业务预期时间、上线时间
- 能力形式、文档地址、备注

**操作步骤**：
1. 点击「新建需求」
2. 填写表单信息
3. 选择多选字段（点击标签选中/取消）
4. 勾选产品PM
5. 点击「提交需求」

### 4.3 编辑需求

**入口**：需求列表 → 操作列 → ✏️ 编辑

**说明**：
- 所有字段均可修改
- 修改后自动更新「登记信息」中的更新时间

### 4.4 删除需求

**入口**：需求列表 → 操作列 → 🗑️ 删除

**规则**：
- ⚠️ **只有「待确认」状态的需求才能删除**
- 其他状态的需求不显示删除按钮
- 删除操作不可恢复

### 4.5 查看需求详情

**入口**：需求列表 → 操作列 → 👁️ 查看

**内容**：
- 需求完整信息
- 审查建议历史

### 4.6 审查建议

**入口**：
- 需求列表 → 操作列 → 💬 审查
- 需求详情页 → 审查建议

**填写内容**：
- 能力形式（复核）

### 4.7 需求排期

**入口**：左侧菜单 → 📅 需求排期

**功能**：
- 为需求分配责任人和时间
- 跟踪开发进度
- 查看甘特图视图

**字段说明**：

| 字段 | 说明 |
|------|------|
| 关联需求 | 选择要排期的需求 |
| 责任人 | 负责该任务的人员 |
| 角色 | 产品/开发/测试/UI |
| 开始时间 | 任务开始日期 |
| 结束时间 | 任务截止日期 |
| 进度 | 完成百分比（0-100%）|
| 状态 | 待排期/进行中/已完成/已延期 |
| 备注 | 补充说明 |

**操作步骤**：
1. 点击「新建排期」
2. 选择关联需求
3. 填写责任人、角色、时间
4. 设置进度和状态
5. 点击「保存」

**甘特图视图**：
- 周视图：按周展示排期
- 月视图：按月展示排期
- 进度条颜色：绿色（已完成）、蓝色（进行中）、红色（已延期）

**筛选功能**：
- 按责任人筛选
- 按角色筛选
- 按状态筛选
- 关联模块（复核）
- 需要应用国家（复核）
- 产品PM（复核）
- 审查建议（限 200 字）

### 4.7 需求统计

**入口**：左侧菜单 → 📊 需求统计

**功能**：
- 按状态统计需求数量
- 按模块分布图表
- 按类型分布图表
- 按国家分布图表
- 上线清单（已发布需求）

**筛选条件**：
- 上线时间、预期时间
- 已应用国家、需要应用国家
- 当前状态、产品PM

### 4.8 导出 Excel

**入口**：需求列表 → 📥 导出 Excel

**说明**：
- 导出当前筛选结果
- 文件格式：CSV（可用 Excel 打开）
- 文件名：需求列表_日期.csv

## 五、状态流转

```
待确认 → 待评审 → 开发中 → 测试中 → 已发布
                ↓
              挂起
                ↓
              已关闭
```

| 状态 | 说明 | 可执行操作 |
|------|------|------------|
| 待确认 | 新建的需求，等待确认 | 编辑、删除、审查 |
| 待评审 | 已确认，等待评审 | 编辑、审查 |
| 开发中 | 正在开发 | 编辑、审查 |
| 测试中 | 开发完成，正在测试 | 编辑、审查 |
| 已发布 | 测试通过，已上线 | 编辑、查看 |
| 已关闭 | 需求关闭 | 查看 |
| 挂起 | 暂时搁置 | 编辑、审查 |

## 六、协作规范

### 6.1 需求提交

1. **谁提出谁登记**：需求提出人负责登记需求
2. **信息完整**：尽量填写所有字段，特别是「需求内容」和「需求价值」
3. **状态设置**：新建需求默认为「待确认」

### 6.2 需求审查

1. **审查人**：组长负责审查
2. **审查内容**：复核能力形式、模块、国家、PM
3. **审查建议**：填写具体意见或建议

### 6.3 状态更新

1. **及时更新**：需求状态变更后及时在平台更新
2. **上线登记**：需求上线后更新「上线时间」和「已应用国家」

### 6.4 数据规范

1. **标题规范**：简洁明了，限 30 字
2. **内容详细**：需求内容要清晰描述问题和期望
3. **国家选择**：使用标准国家名称

## 七、常见问题

### Q1: 无法登录？
- 检查用户名和密码是否正确
- 账号有效期为 2 个月，过期需重新登录
- 联系管理员重置密码

### Q2: 需求无法删除？
- 只有「待确认」状态的需求才能删除
- 其他状态需要先修改状态为「待确认」

### Q3: 多选字段如何操作？
- 点击标签选中（变紫色）
- 再次点击取消选中（变回灰色）

### Q4: 数据会丢失吗？
- 数据存储在云端，不会丢失
- 清除浏览器缓存不影响数据

### Q5: 如何导出数据？
- 在需求列表页点击「导出 Excel」
- 导出的是当前筛选结果

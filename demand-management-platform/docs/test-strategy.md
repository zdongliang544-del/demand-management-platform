# 分销团队需求管理平台 - 测试策略文档

> 编写人：严过关（QA 工程师）  
> 编写日期：2026-06-25  
> 平台地址：demand.sevenway.top  
> 技术架构：纯静态 SPA（单 HTML 文件），localStorage + CloudBase 双存储

---

## 一、测试范围与目标

### 1.1 测试范围

| 模块 | 功能点 | 优先级 |
|------|--------|--------|
| 登录/会话 | 用户认证、会话持久化、登出 | P0 |
| 需求列表 | CRUD、筛选、导出、详情、审查建议 | P0 |
| 需求排期 | 排期CRUD、甘特图、卡点标记、超时计算 | P0 |
| 人员看板 | 数据聚合、角色筛选、统计准确性 | P1 |
| 我的任务 | 当前用户任务、状态分组、Markdown导出 | P1 |
| 报告生成 | 周报/月报、Markdown导出 | P1 |
| 需求统计 | 图表数据准确性、筛选联动 | P1 |
| 数据导入 | 语雀数据批量导入 | P2 |
| 响应式/UI | 布局、交互反馈、边界展示 | P2 |

### 1.2 测试目标

1. 验证所有功能模块按需求正常工作
2. 确保数据一致性（localStorage ↔ CloudBase）
3. 验证边界条件和异常处理的健壮性
4. 确保用户操作体验流畅，无死循环或卡死
5. 验证导出功能（Excel/Markdown）内容完整性

### 1.3 测试环境

- **浏览器**：Chrome 120+、Safari 17+、Firefox 120+、Edge 120+
- **存储模式**：localStorage 模式（主）、CloudBase 模式（辅）
- **测试数据**：71条预置迁移数据 + 手动创建数据
- **用户账号**：6个账号（2个组长 + 4个组员）

## 二、功能测试用例

### 2.1 登录/会话模块

#### TC-AUTH-001：正常登录
- **前置条件**：未登录状态
- **步骤**：输入正确用户名和密码，点击登录
- **预期**：跳转到需求列表页，侧边栏显示用户名和角色，Toast提示"欢迎回来"
- **覆盖**：6个账号逐一验证

#### TC-AUTH-002：错误凭据登录
- **步骤**：输入错误用户名或密码
- **预期**：Toast提示"用户名或密码错误"，停留在登录页

#### TC-AUTH-003：会话持久化（2个月有效期）
- **步骤**：登录后刷新页面
- **预期**：自动进入主页面，无需重新登录
- **补充**：手动修改 localStorage 中 session 的 expiry 为过去时间，刷新后应回到登录页

#### TC-AUTH-004：退出登录
- **步骤**：点击"退出登录"，确认
- **预期**：清除会话，回到登录页

#### TC-AUTH-005：退出登录取消
- **步骤**：点击"退出登录"，取消
- **预期**：留在当前页面，会话不变

#### TC-AUTH-006：空用户名/密码提交
- **步骤**：不填用户名或密码，点击登录
- **预期**：浏览器原生 required 校验阻止提交

## 九、已知风险点

| # | 风险描述 | 严重程度 | 建议 |
|---|----------|----------|------|
| 1 | 密码明文存储在前端代码中（`USERS` 对象） | 高 | 建议后端接口认证，或至少混淆处理 |
| 2 | localStorage 无容量上限保护，大量数据可能溢出 | 中 | 监控数据量，考虑分页加载 |
| 3 | CloudBase 保存采用"先全删再全写"策略，并发场景可能丢数据 | 高 | 改用单条 upsert 操作 |
| 4 | 全部代码集中在单个 HTML 文件（~190KB），可维护性差 | 中 | 建议模块化拆分 |
| 5 | CSV导出的日期字段可能含逗号导致列错位 | 低 | 已做引号包裹，但需验证含逗号日期场景 |
| 6 | 甘特图无横轴滚动时日期标签可能重叠 | 低 | 当前实现 min-width 兜底，大量排期时体验待验证 |
| 7 | 排期与需求未做级联删除约束 | 中 | 删除需求后排期成为孤立数据 |
| 8 | 无操作日志/审计功能 | 低 | 建议后续增加变更历史追踪 |
| 9 | "导入语雀数据"按钮在侧边栏常驻，可能误触 | 低 | 有确认弹窗保护，风险可控 |
| 10 | 卡点天数计算依赖客户端时间，时区差异可能影响 | 低 | 当前团队在同一时区，暂无影响 |
